# Prototype demo data — 100% synthetic

**Every record in this folder is fictional.** Names, sessions, GPS/load numbers,
lift and Perch values, and availability entries are computer-generated. There is
**no real athlete data** here — no personal, contact, medical, or identifiable
information of any kind. This is prototype demonstration data only.

## What this is

The current dashboard is a **client-only prototype** that runs entirely on this
synthetic dataset (there is no backend yet). These files are what the app fetches
at runtime under `/dev-data/`:

| File | Used by |
| --- | --- |
| `canonical.json` | Every dashboard page (the season dataset) |
| `fixtures/*.csv` | The Import page's "load a sample export" feature |
| `fixtures.json` | The list of available sample CSVs (static index) |

The app loads the **same** files locally (`npm run dev`), in the GitHub Pages
build, and in fresh clones — the Vite dev server falls back to this folder when a
freshly generated dataset (`seed/output/current/`) is not present.

## Deterministic & reproducible

The dataset is generated by the seed generator and is fully deterministic:

```bash
npm run seed:generate      # season 2026, seed 20260801 (the committed defaults)
```

- `generatorVersion`: 1.0.0 · `seed`: 20260801 · season: 2026-08-10 → 2026-12-06
- 25 fictional athletes · 139 sessions

Regenerating with the same defaults reproduces this dataset exactly. To refresh the
committed demo data, run the generator and copy `seed/output/current/canonical.json`
and `seed/output/current/fixtures/` here, then regenerate `fixtures.json`.

## Do not add real data

Real athlete data must never enter this repository (`.gitignore` and the
build-time environment guard enforce this). Keep this folder synthetic.
